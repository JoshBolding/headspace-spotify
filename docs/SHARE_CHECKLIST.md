# Share Checklist

Use this before posting the repo in Discord, GitHub, or a builder community.

## Required

- Run `npm run build`.
- Confirm `.env` is not tracked.
- Confirm README setup instructions are accurate.
- Confirm `docs/SPOTIFY_SETUP.md` matches the current app redirect URI.
- Open the app and verify Spotify sign-in still works.
- Capture one current screenshot or short GIF/video from the latest build.
- Decide whether to show the face-alive Easter egg or keep it as a hidden detail.

## Nice To Have

- Add a short demo GIF near the top of `README.md`.
- Add a tagged GitHub release once there is a packaged build.
- Add fresh screenshots to `docs/screenshots`.
- Add repo topics on GitHub: `electron`, `spotify`, `desktop-app`, `media-player`, `nostalgia`.

## Discord Post

Use `docs/DISCORD_SHARE.md` as the starting draft. Keep the post honest: this is a builder/demo project, not a fully packaged product yet.
